## Step 2 — Configure Vault

Open a shell inside the container:

    docker exec -it vault-dev sh

Then run the following commands one at a time:

    export VAULT_ADDR=http://127.0.0.1:8200
    export VAULT_TOKEN=dev-root-token

    vault secrets enable transit

    vault write -f transit/keys/session-keys type=aes256-gcm96

    vault auth enable approle

    vault policy write wallet-agent - <<EOF
    path "transit/encrypt/session-keys" { capabilities = ["update"] }
    path "transit/decrypt/session-keys" { capabilities = ["update"] }
    EOF

    vault write auth/approle/role/wallet-agent \
        token_policies="wallet-agent" \
        token_ttl=1h \
        token_max_ttl=4h

    vault read auth/approle/role/wallet-agent/role-id

    vault write -f auth/approle/role/wallet-agent/secret-id

Copy the role_id and secret_id from the output and save them to .env as:

    VAULT_ROLE_ID=<role_id>
    VAULT_SECRET_ID=<secret_id>
    VAULT_ADDR=http://127.0.0.1:8200

Also save the secret_id_accessor somewhere safe — you will need it to revoke
or inspect the secret_id later without needing the secret itself.

Type 'exit' to leave the container shell when done.

NOTE: This must be re-run every time the Vault container restarts, because
dev mode stores everything in memory and resets on restart.
